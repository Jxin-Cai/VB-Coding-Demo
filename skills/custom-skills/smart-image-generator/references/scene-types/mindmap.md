# 场景类型：思维导图

## 用途

创建清晰的思维导图可视化，要求：
- 展示概念之间的层级关系
- 显示主题和子主题的连接
- 辅助头脑风暴和概念组织
- 支持学习和记忆

## 检测关键词

- "思维导图"、"mind map"、"概念图"
- "分支"、"主题"、"子主题"
- "中心思想"、"核心概念"
- 层级结构指示词："主"、"次"、"分支"
- 关联词："连接"、"关联"、"关系"

## 提示词架构

### 必需组件

1. **中心主题**：核心概念或主题
2. **主分支**：从中心延伸的主要想法（3-7个）
3. **子分支**：主分支下的细节（可选）
4. **连接线**：显示关系的线条
5. **视觉层级**：通过大小、颜色、距离体现

### 思维导图类型

**经典放射型**：
```
         分支1
          ↗
中心主题 ← 分支2
          ↘
         分支3
```

**水平层级型**：
```
分支1 ─┐
分支2 ─┤─ 中心主题
分支3 ─┘
```

**树形结构**：
```
           中心
          ↙  ↓  ↘
      分支1 分支2 分支3
       ↓    ↓↓   ↓
     子项 子项子项 子项
```

```markdown

- 位置：画布中心
- 视觉：{圆形/椭圆/矩形}
- 强调：{最大字体/高亮颜色}

主分支：
1. {主题1}
   - 关键词：{简短描述}
   - 颜色：{分支颜色}
   - 子分支：{子主题列表或"无"}
   
2. {主题2}
   - 关键词：{简短描述}
   - 颜色：{分支颜色}
   - 子分支：{子主题列表或"无"}

3. {主题3}
   - 关键词：{简短描述}
   - 颜色：{分支颜色}
   - 子分支：{子主题列表或"无"}

连接线样式：
- 类型：{曲线/直线/有机线条}
- 粗细：根据层级递减
- 颜色：{继承父分支/统一/渐变}

- 中心主题：最大、最醒目
- 主分支：中等大小、色彩编码
- 子分支：较小、同色系但更浅

节点样式：
- 中心：{形状、填充色、边框}

- 中心：{醒目主色}

- 背景：{浅色、不抢眼}

布局与构图：
- 平衡：各分支视觉重量均衡
- 间距：避免拥挤或过疏

- 背景：{纯色/微妙纹理}

{任何风格特定的指示}
```

## 最佳实践

### 应该做的

- ✅ 中心主题清晰突出
- ✅ 使用简短关键词（不超过3-5个字）
- ✅ 色彩编码不同主题分支
- ✅ 保持视觉平衡（分支均匀分布）
- ✅ 连接线清晰可见
- ✅ 层级关系一目了然

### 不应该做的

- ❌ 文本过多（思维导图重在简洁）
- ❌ 分支过密或重叠
- ❌ 使用过多层级（建议≤3层）
- ❌ 颜色杂乱无章
- ❌ 忽视视觉层级
- ❌ 连接线混乱交叉

## 示例

### 示例 1：学习主题思维导图

**输入**："学习 Python：基础语法、数据结构、函数、面向对象、实践项目"

**提示词**：
```
场景：思维导图
风格：obsidian
布局：放射型

中心主题：
- 概念：Python 学习
- 视觉：圆形，紫色填充
- 文本：粗体、白色

主分支：
1. 基础语法
   - 关键词：变量、运算符、控制流
   - 颜色：蓝色
   - 位置：右上（2点钟方向）
   - 子分支：if/for/while
   
2. 数据结构
   - 关键词：列表、字典、集合
   - 颜色：绿色
   - 位置：右侧（3点钟）
   - 子分支：list/dict/set/tuple

3. 函数
   - 关键词：定义、参数、返回值
   - 颜色：橙色
   - 位置：右下（4点钟）
   - 子分支：def/lambda/装饰器

4. 面向对象
   - 关键词：类、继承、多态
   - 颜色：红色
   - 位置：左下（8点钟）
   - 子分支：class/继承/方法

5. 实践项目
   - 关键词：爬虫、数据分析、Web
   - 颜色：青色
   - 位置：左上（10点钟）
   - 子分支：requests/pandas/Flask

连接线：
- 有机曲线，从中心辐射
- 继承父分支颜色
- 主分支粗、子分支细

背景：白色或浅灰
手绘质感：微妙
```

## 故障排除

### 分支过多

**问题**：超过7个主分支，视觉混乱

**解决方案**：
1. 合并相似主题
2. 创建更高层次的分类
3. 将某些主分支降级为子分支
4. 考虑创建多个关联思维导图

### 层级过深

**问题**：超过3层，难以理解

**解决方案**：
1. 简化层级，只保留关键层次
2. 为深层内容创建独立思维导图
3. 使用更概括的描述
4. 询问用户哪些可以省略

### 文本过长

**问题**：节点文本超过5个字

**解决方案**：
1. 使用关键词而非完整句子
2. 使用缩写或符号
3. 将长文本拆分为多个节点
4. 在子分支中展开详细信息

## 场景特定的协作问题

在与用户讨论思维导图提示词时，询问：

1. **结构偏好**：
   - "希望使用放射型还是层级型布局？"
   - "需要包含子分支吗？"

2. **视觉风格**：
   - "喜欢多彩还是单色系？"
   - "需要手绘还是数字精确风格？"

3. **详细程度**：
   - "每个主题需要展开到什么程度？"
   - "哪些分支最重要需要强调？"

4. **使用场景**：
   - "这是用于演示还是个人笔记？"
   - "需要适配特定尺寸吗？"
