# 场景类型：笔记插图

## 用途

为笔记和文档创建辅助性插图，要求：

- 简化复杂概念的视觉表达
- 增强笔记的可读性和记忆点
- 提供快速视觉参考
- 支持学习和理解

## 检测关键词

- "笔记"、"note"、"记录"
- "插图"、"示意图"、"图解"
- "简化"、"说明"、"展示"
- 教学词汇："理解"、"学习"、"掌握"
- 参考指示："如图"、"见下图"

## 提示词架构

### 必需组件

1. **核心概念**：要解释的主要内容
2. **视觉隐喻**：帮助理解的象征或比喻
3. **关键元素**：2-5个要点
4. **标注**：简短文字说明（可选）
5. **简洁性**：避免过度复杂

### 笔记插图类型

**概念说明图**：

- 用途：解释抽象概念
- 特点：简单象征、清晰标注

**对比图**：

- 用途：展示差异或选择
- 特点：并列布局、清晰对比

**流程简图**：

- 用途：快速展示步骤
- 特点：简化流程、关键步骤

**关系图**：

- 用途：显示元素间关系
- 特点：连接线、层级或网络

**示例图**：

- 用途：提供具体例子
- 特点：实际场景、真实元素

## 模板结构

```markdown

风格：{风格名称}

插图目的：

- 主题：{要解释的核心概念}
- 受众：{学生/初学者/专业人士}
- 复杂度：{简单/中等}

核心视觉：

- 主元素：{中心图形或象征}
  - 描述：{元素的具体表现}
  - 位置：{画面中的位置}
  - 强调：{如何突出}

- 辅助元素：{2-4个关键点}
  - 元素1：{描述}
    - 关系：{与主元素的关系}
  - 元素2：{描述}
  - 元素3：{描述}

标注与说明：

- 主标题："{核心概念名称}"
- 子标注：{关键点的简短文字}
  - 位置：靠近对应元素
  - 风格：小字、箭头指示
- 是否需要详细解释：{是/否/最小}

视觉风格：

- 线条：{简洁线条艺术/手绘草图}
- 颜色：{最少用色/单色+强调/适度彩色}

{任何特定要求}
```

## 最佳实践

### 应该做的

- ✅ 保持简洁，避免过度细节
- ✅ 一图一个核心概念
- ✅ 使用直观的视觉隐喻
- ✅ 标注清晰简短
- ✅ 与笔记内容相呼应
- ✅ 易于快速理解

### 不应该做的

- ❌ 过度复杂化
- ❌ 试图在一张图中包含太多
- ❌ 使用晦涩的象征
- ❌ 忽略目标受众
- ❌ 标注文字过多
- ❌ 装饰性大于功能性

## 示例

### 示例 1：HTTP 请求流程笔记插图

**输入**："HTTP 请求：客户端发送请求 → 服务器处理 → 返回响应"

**提示词**：

```
场景：笔记插图
风格：obsidian
类型：流程简图

插图目的：
- 主题：HTTP 请求响应流程
- 受众：初学者
- 复杂度：简单

核心视觉：
- 主元素：三个阶段的简化图示
  - 客户端：笔记本电脑图标
  - 服务器：服务器图标
  - 箭头：显示数据流向

流程：
1. [客户端 💻]
   ↓
   请求 →

2. [服务器 🖥️]
   处理...
   ↓
   响应 ←

3. [客户端 💻]
   显示结果

标注：
- "1. 发送 GET /api/data"（客户端旁）
- "2. 查询数据库"（服务器旁）
- "3. 返回 JSON"（响应箭头上）

视觉风格：
- 线条：简洁黑色线条
- 图标：极简几何图形
- 颜色：黑白 + 单色强调（蓝色箭头）
- 背景：纯白

布局：水平流程，从左到右
留白：元素间距宽松
```

### 示例 2：MVC 架构笔记插图

**输入**："MVC模式：Model(数据)、View(界面)、Controller(控制器)的关系"

**提示词**：

```
场景：笔记插图
风格：notion
类型：关系图

插图目的：
- 主题：MVC 架构模式
- 受众：编程学习者

核心视觉：
- 三个组件盒子，显示相互关系

组件布局：
    [View]
      ↕
 [Controller]
      ↕
    [Model]

详细元素：
1. Model 盒子（底部）
   - 圆角矩形，绿色
   - 图标：数据库符号
   - 标注："数据和业务逻辑"

2. Controller 盒子（中间）
   - 圆角矩形，橙色
   - 图标：齿轮
   - 标注："处理请求"

3. View 盒子（顶部）
   - 圆角矩形，蓝色
   - 图标：显示器
   - 标注："用户界面"

连接箭头：
- Model ↔ Controller：双向，"读写数据"
- Controller ↔ View：双向，"更新显示"
- View → Controller：单向虚线，"用户操作"

视觉风格：
- 简洁数字图形
- 柔和颜色填充
- 清晰标注
- 现代SaaS风格

背景：浅灰
```

### 示例 3：时间管理矩阵笔记插图

**输入**："时间管理：重要紧急、重要不紧急、不重要紧急、不重要不紧急"

**提示词**：

```
场景：笔记插图
风格：minimal
类型：概念说明图

插图目的：
- 主题：艾森豪威尔时间管理矩阵
- 受众：效率提升学习者

核心视觉：
- 2x2 矩阵格子

布局：
         紧急    |  不紧急
    ────────────┼────────────
重要  │ ① 危机  │ ② 计划
     │  ⚠️     │  📅
    ────────────┼────────────
不   │ ③ 干扰  │ ④ 浪费
重要  │  📧     │  📱
    ────────────┴────────────

四个象限：
1. 重要且紧急（左上，红色）
   - 标签："① 马上做"
   - 示例：⚠️ 危机、截止日

2. 重要不紧急（右上，绿色）
   - 标签："② 计划做"
   - 示例：📅 规划、学习

3. 不重要紧急（左下，黄色）
   - 标签："③ 委托"
   - 示例：📧 部分邮件

4. 不重要不紧急（右下，灰色）
   - 标签："④ 不做"
   - 示例：📱 刷手机

视觉风格：
- 极简线条
- 单色 + 四色区分
- 简洁图标
- 最少文字

背景：纯白
强调：清晰的象限边界
```

## 故障排除

### 概念过于抽象

**问题**：难以找到合适的视觉表达

**解决方案**：

1. 使用类比和隐喻
2. 将抽象具体化（如"数据流" → 水流）
3. 用简单几何形状代表
4. 询问用户："您会如何向朋友解释这个？"

### 信息量过大

**问题**：笔记内容太多，一图难以包含

**解决方案**：

1. 拆分为多张插图
2. 只展示核心要点
3. 创建系列插图（1/3, 2/3, 3/3）
4. 使用概览图 + 详细分图

### 与笔记风格不符

**问题**：插图风格与笔记整体风格不匹配

**解决方案**：

1. 询问用户笔记风格（手写/数字/简洁/丰富）
2. 调整插图风格匹配
3. 提供多个风格选项
4. 使用与笔记工具风格一致的视觉语言

## 场景特定的协作问题

在与用户讨论笔记插图提示词时，询问：

1. **插图类型**：
   - "需要解释概念、展示流程，还是对比差异？"
   - "是配图说明还是独立参考？"

2. **详细程度**：
   - "需要多详细？快速理解还是深入学习？"
   - "是否需要文字标注？"

3. **视觉风格**：
   - "您的笔记是什么风格？（手写/数字）"
   - "喜欢简洁还是更丰富的插图？"

4. **使用场景**：
   - "这是给自己看还是分享给他人？"
   - "需要配合什么笔记应用？（Obsidian/Notion/纸质）"
