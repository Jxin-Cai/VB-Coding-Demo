# 场景类型：文章封面

## 用途

为文章、博客帖子和书面内容创建引人注目的封面图，要求：

- 在单个视觉中捕捉文章的精髓
- 吸引读者点击和阅读
- 适合作为缩略图和主图
- 反映文章的基调和信息

## 检测关键词

- "封面"、"头图"、"hero图"、"首图"
- "文章"、"帖子"、"博客"
- 存在明确的文章标题
- 摘要或概述部分
- 作者/日期元数据

## 提示词架构

### 必需组件

1. **标题/主标语**（如果不是 --no-title）：
   - 从 H1 或 frontmatter 标题中提取
   - 保持在 8-10 个字以内以产生影响力
   - 使用钩子：数字、问题、对比

2. **核心信息**：
   - 用 1-2 句话提炼主要要点
   - 识别关键洞察或收益
   - 观众应该记住什么？

3. **视觉隐喻**：
   - 主题的象征性表现
   - 避免字面插图
   - 尽可能使用通用符号

4. **构图**：
   - 布局：居中、不对称、分屏
   - 焦点：眼睛应该首先落在哪里
   - 视觉层次：标题 → 图像 → 辅助元素

### 风格适配

**针对不同内容类型**：

- **技术文章**：蓝图或图表式元素
- **个人故事**：温暖、亲密的意象
- **产品发布**：大胆、商业化感觉
- **教育内容**：简洁、有组织的布局
- **观点文章**：动态、有主见的构图

### 宽高比指导

- **16:9**：博客平台标准，视频缩略图
- **2.35:1**：电影感、戏剧性、编辑风格
- **1:1**：社交媒体（Instagram、Twitter 卡片）
- **自定义**：匹配特定平台要求

## 模板结构

```markdown
场景：文章封面图
风格：{风格名称}
宽高比：{比例}

文章上下文：

- 标题：{提取的标题}
- 核心主题：{主要主题}
- 基调：{严肃/有趣/鼓舞/教育性}
- 目标受众：{谁会阅读这个}

视觉概念：

- 主视觉：{中心象征元素}
  - 类型：{插图/图标/抽象}
  - 位置：{居中/左/右}

  - 颜色：{基础颜色和处理}

- 标题文本："{确切标题文本}"
- 字体处理：{粗体/优雅/手写/几何}
- 位置：{顶部/中心/底部}
```

## 最佳实践

### 应该做的

- ✅ 使用隐喻和符号，而非字面表现
- ✅ 保持视觉复杂度低以便缩略图可读
- ✅ 确保标题文本在小尺寸时可读
- ✅ 匹配视觉基调与文章基调
- ✅ 有效使用负空间
- ✅ 考虑色彩心理学

### 不应该做的

- ❌ 用太多元素拥挤画面
- ❌ 使用通用库存照片美学
- ❌ 标题文本太小或太装饰
- ❌ 忽视文章实际内容
- ❌ 无目的地使用冲突颜色
- ❌ 过于模仿其他封面

## 示例

### 示例 1：技术教程

**文章**："使用 Kubernetes 构建可扩展微服务"

**提示词**：

```
场景：文章封面
风格：blueprint（蓝图）
宽高比：16:9

视觉：等轴测视图中漂浮的互连几何块
- 每个块代表一个微服务
- 它们之间有清晰的连接线
- 蓝图风格的网格背景
- 技术标注标签

标题："扩展微服务"
位置：左上角
处理：工程风格字体

颜色：
- 主色：深蓝（#1A4D8F）
- 网格：浅蓝（#C8E6F5）
- 强调：橙色（#FF6B35）
```

### 示例 2：个人成长故事

**文章**："我如何克服冒名顶替综合症"

**提示词**：

```
场景：文章封面
风格：warm（温暖）
宽高比：16:9

视觉：单人站在山峰顶，身后日出
- 剪影风格
- 温暖的渐变天空（橙到粉）
- 身后有小脚印痕迹
- 微妙的成就符号（小旗、背包）

标题："克服冒名顶替综合症"
位置：中心，叠加在天空上
处理：友好的手写风格

颜色：
- 主色：温暖橙（#FF8C42）
- 次色：柔和粉（#FFB4A2）
- 强调：深紫（阴影）（#5D3F6A）
```

### 示例 3：数据分析文章

**文章**："理解用户留存指标"

**提示词**：

```
场景：文章封面
风格：editorial-infographic（编辑信息图）
宽高比：2.35:1

视觉：风格化条形图转化为生长的植物
- 左侧：几何条形/数字
- 中心：过渡区
- 右侧：有机植物/生长
- 代表用户的简单图标

标题："留存指标"
位置：下三分之一
处理：粗体无衬线字体

颜色：
- 主色：数据蓝（#0066CC）
- 生长：鲜绿（#00C851）
- 背景：米白（#F5F5F5）
```

## 故障排除

### 标题无法提取

**问题**：文章没有明确标题或 H1

**解决方案**：

1. 检查 frontmatter 的 `title:` 字段
2. 扫描前 100 个词寻找类似标题的文本
3. 询问用户："封面标题应该是什么？"
4. 如果适当，回退到 `--no-title`

### 基调模糊

**问题**：无法确定文章是严肃还是有趣

**解决方案**：

1. 检查写作风格（正式 vs 随意语言）
2. 寻找幽默、表情符号、随意措辞
3. 如果不清楚默认为中性/专业
4. 询问用户基调偏好

### 主题过于泛泛

**问题**：文章主题太宽泛或通用（"AI"、"技术"）

**解决方案**：

1. 扫描具体示例或角度
2. 识别独特视角或钩子
3. 关注提到的子主题或案例研究
4. 询问用户："这篇文章的独特之处是什么？"

## 场景特定的协作问题

在与用户讨论封面提示词时，询问：

1. **视觉焦点**：
   - "封面应该更概念化还是更字面化？"
   - "您心中有特定的符号或隐喻吗？"

2. **标题处理**：
   - "应该包含完整标题还是缩短？"
   - "标题应该唤起什么情感？"

3. **受众对齐**：
   - "这是给专家还是初学者看的？"
   - "应该是邀请式的还是权威式的？"

4. **平台考虑**：
   - "主要在哪里使用？"
   - "需要作为小缩略图工作吗？"
